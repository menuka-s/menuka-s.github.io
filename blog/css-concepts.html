Week 3

CSS-concepts

It's the third week of DBC, and that means it's time for another blog post! This week I was asked to look at a series of questions and pick one to answer. The question I chose was: What are the best practices associated with using classes vs. IDs?

Both classes and IDs are used to help you designate certain sections to get a certain CSS styling added. Basically both are just names that you can refer to in CSS. This is helpful when you have multiple sections in your HTML that have a similar code structure, but need different CSS styling.

From some research I've done, it looks like IDs are mainly used if need to specify some section of HTML once. Classes are used if you need to refer to this section over and over. So when you are planning out your HTML code and thinking about potential CSS styling, it is a good idea to ask yourself if you need to add the same CSS styling to multiple sections.

Both IDs and classes can exist in the same files, but it appears that ID selectors do get a higher specificity than class selectors. So it is harder to overwrite an ID with a class in subsequent CSS files.
