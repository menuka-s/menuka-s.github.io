Week 6

Ruby Classes
12/20/15

Time to dive into a staple of object-orientated programming: Classes.

Classes are a form of objects that are specifically created to represent and do something. When these objects are created they will have special variables defined and exclusive methods just for it. A user can initialize as many of objects of any specific classes as they want.

So lets use a real world example to help explain what classes are. Let's use people as an example. First of all, lets show the barebones of a person class, and one of the first things that everyone has is a name. We can create a new class for a specific person as so: 


          class Person
            def initialize(name)
              @name = name
            end
          end

This is the basic barebones of a person class. Inside the class we have a initialize method which runs when a new class is created. So if we create a new person with person1 = Person.new("Menuka"), this creates a new person class called person1. With our initialization method, our person will automatically have an instance variable, @name, store the given name, Menuka.

If we wanted, we could pass in other data that can be used as instance varibles too. Such as age or gender. We can also have the initialize method automatically create instance variables too without data being passed. Here is a more worked out class example: 


          class Person
            def initialize(name, age, gender)
              @name = name
              @age = age
              @gender = gender
              @is_alive = true
            end
          end

If we wanted to access person1's age, we need to add an extra line to our class code to add an attribute reader. We can add one with attr_reader :age. This lets us use person1.age to get that person's age.

Classes also allow us to create methods that just our person class can use. Let's create a greeting method that greets us: 


            class Person

            attr_reader :age
            attr_reader :name
            attr_reader :gender

            def initialize(name, age, gender)
              @name = name
              @age = age
              @gender = gender
              @is_alive = true
            end

            def greeting
              puts "Hello World, my name is #{@name}"
            end
          end

As you can see we now have a method called greeting that will print some output (plus it includes one of our inheritance variables). We could call person1.greeting which would output "Hello World, my name is Menuka".

Now we have a basic class that can create a new object when called and has its own special method! Some more advance things we can look into later include inheritance and other class-specific tricks.
