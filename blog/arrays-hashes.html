Week 4

Arrays and hashes
12/20/15

Week 4 and the holidays have quickly gone by. Time for another blog post. This week I'll be talking about two simple data structures: arrays and hashes.

Arrays are an ordered container for objects or data. Arrays can hold many types of information such as integers, strings, or even other arrays. Each piece of information in an array is indexed in the container with a number, which allows an easy way to find data at certain positions in the array. You can easily add things to the end of an array, but it may be a little harder to add items to specific positions within an array without using built-in methods. Removing items poses similar problems.

An example array would be:

my_array = [“variable 1”, 2, variable3]

where each element in the square brackets is some sort of data. This array is 3 elements big. “variable 1” is in position 0 of the array.

Hashes are an unordered container that uses unique keys to match with values. This key that always points to the value assigned to it. So if you want to find a something specific in a hash, and you know the key, then you can easily search the hash for that key. Like arrays, they are not restricted in what they can store.

An example hash would be:

my_hash = { “key1” => value1, “key2 => “value2”}

where “key1” and “key2” are they unique keys and they are only linked to value1 and “value2” respectively.

There are different situations where you would use an array or a hash. Hashes function as simple databases, so it would be easy to use one if you want to store large sets of data that you can make keys for. Arrays are not efficient to traverse through so they aren’t the best for large sets of data. If you don’t know the index of a specific element you want, you may have to traverse through the entire array to find it.
